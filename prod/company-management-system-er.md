# Company Management System – Entity Relationship Model

Entity Relationship model for the Company Management System (CMS) in Global Payroll.
Covers payroll configuration, tax registrations, benefits, policies, and financial accounts. Organisational hierarchy entities (legal entity, business unit, department, cost center, work location, job position) are defined but deferred as optional future additions.

---

| # | Entity | Parent / FK to | Purpose |
|---|--------|---------------|---------|

## Core entity details

### 1. COMPANY

Top-level registered entity and customer. Each jurisdiction registration (e.g. Uber India, Uber US) is a separate COMPANY with its own KYB, data isolation, local regulation adherence, and GDPR scope. A single admin user can be associated with multiple companies. Field mapping follows the `Customer` schema in the OpenAPI spec.

| Column | Type | Constraint | Description |
|--------|------|-----------|-------------|
| company_id | UUID | PK | Maps to `customerId`. Generated on create. |
| company_name | VARCHAR(250) | NOT NULL | Display name of the company |
| legal_entity_name | VARCHAR(250) | NOT NULL | Registered legal entity name |
| tax_id | VARCHAR(100) | NOT NULL | Tax identification number (format varies by country) |
| registration_number | VARCHAR(100) | | Business / company registration number |
| address_line_1 | VARCHAR(250) | | Registered address: primary street address |
| address_line_2 | VARCHAR(250) | | Registered address: secondary line (suite, unit, etc.) |
| city | VARCHAR(100) | | Registered address: city |
| state_province | VARCHAR(100) | | Registered address: state, province, or region |
| postal_code | VARCHAR(20) | | Registered address: postal / ZIP code |
| country_code | CHAR(2) | | Registered address: ISO 3166-1 alpha-2 |
| kyb_status | VARCHAR(20) | NOT NULL | NOT_STARTED, IN_PROGRESS, APPROVED, REJECTED. Managed by KYB workflow. |
| status | VARCHAR(20) | NOT NULL | ACTIVE, INACTIVE, PENDING |
| payroll_start_date | DATE | | Date when payroll processing begins for this customer |
| created_at | TIMESTAMPTZ | NOT NULL | Read-only, set on create |
| updated_at | TIMESTAMPTZ | NOT NULL | Read-only, updated on every write |

### 11. OVERTIME_POLICY_RULE

One row per (context + tier): when overtime is worked (weekday, weekend, national holiday) and optional hour tier (e.g. first 5h at 1.5x, above 5h at 2x). Payroll evaluates rules by context and applied hours to pick the multiplier.

| Column | Type | Constraint | Description |
|--------|------|-----------|-------------|
| overtime_rule_id | UUID | PK | |
| overtime_policy_id | UUID | FK → OVERTIME_POLICY | |
| apply_to | VARCHAR(30) | NOT NULL | WEEKDAY, WEEKEND, NATIONAL_HOLIDAY, REST_DAY — when this rule applies. |
| tier_order | SMALLINT | NOT NULL | 1-based order within same apply_to (1 = first tier, 2 = next, etc.). |
| max_hours_in_tier | DECIMAL(5,2) | | Cap for this tier (e.g. 5 = first 5 hours); NULL = all remaining hours in that context. |
| multiplier | DECIMAL(4,2) | NOT NULL | Rate applied (e.g. 1.50, 2.00). |
| created_at | TIMESTAMPTZ | NOT NULL | |
| updated_at | TIMESTAMPTZ | NOT NULL | |

Example: WEEKDAY tier 1 max 5h → 1.5x; WEEKDAY tier 2 NULL → 2x; WEEKEND tier 1 NULL → 1.5x; NATIONAL_HOLIDAY tier 1 NULL → 2x.

## Relationship summary

Relationship labels in the Mermaid ER diagram use two verbs for consistency:

- **has** — Parent owns or aggregates the child (e.g. COMPANY has PAY_GROUP, PAY_GROUP has PAY_SCHEDULE).
- **contains** — Parent is a container for list/detail rows (e.g. OVERTIME_POLICY contains OVERTIME_POLICY_RULE, HOLIDAY_CALENDAR contains HOLIDAY_CALENDAR_ENTRY).
